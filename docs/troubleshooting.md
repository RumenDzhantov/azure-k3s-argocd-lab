# Troubleshooting Notes

This document provides a high-level overview of troubleshooting areas explored in the Azure k3s lab.

## Main Troubleshooting Areas

The lab has been used to practice troubleshooting in areas such as:
- VM connectivity
- Linux service status
- k3s deployment issues
- ingress behavior
- service exposure
- DNS and routing checks
- Argo CD sync and access issues

## Skills Practiced

This troubleshooting work helps build practical experience in:
- reading service status information
- checking logs
- verifying network paths
- testing service reachability
- isolating configuration problems
- documenting technical findings

## Value

Troubleshooting is an important part of real infrastructure work, and this lab is used to strengthen that practical skill set.

## Notes

The documentation in this repository is intentionally kept at a high level and excludes sensitive operational details.
